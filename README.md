# A+ Assistant

A+ Assistant 是一款 Windows 桌面行情助手，基于 Tauri 2、Vue 3、TypeScript 和 Rust 构建。当前最新正式版为 `1.0.2`，支持股票与场外开放式基金自选、实时刷新、拖拽排序、持仓盈亏、账户汇总、窗口置顶、自启动、托盘常驻和升级安装。

## 最新正式版

- 版本：`1.0.2`
- 应用名称：`A+助手`
- 支持平台：Windows 10 / Windows 11 x64
- 安装包格式：NSIS x64
- 安装包路径：`src-tauri/target/release/bundle/nsis/A+助手_1.0.2_x64-setup.exe`
- 默认安装目录：优先使用 `D:\A+助手`，无 D 盘时使用用户本地应用目录
- 升级安装：检测到旧版本时显示中文升级提示，并支持保留自选、排序、设置和持仓数据
- 图标策略：安装包图标、程序图标、窗口图标和托盘图标统一使用项目默认图标资源

## 1.0.2 更新内容

- 新增 `A+Stock Assistant` / `A+Fund Assistant` 双模式，点击标题切换股票和基金。
- 新增场外开放式基金支持，基金可搜索、添加、删除、刷新、排序和持久化。
- 股票和基金分别保存自选列表、排序和持仓数据。
- 股票支持右键录入成本价和股数，计算市值、成本、盈亏和盈亏比例。
- 基金支持右键录入持有金额和收益，计算成本、盈亏和盈亏比例。
- 股票和基金在录入有效持仓后，顶部显示账户资金、当日预计收益和账户总收益；未录入持仓时保持原列表展示。
- 修复股票自动刷新偶发返回空数据时列表显示“暂无股票”的问题，刷新异常或空响应时保留旧行情。
- 优化基金列表展示，直接显示基金名称、代码、估算涨跌幅和估算净值。
- 优化自选列表拖拽排序，拖动仅从左侧手柄触发，并保留右键排序作为备选。
- 删除按钮只在悬停、焦点或选中状态出现，默认状态下行情数据靠右显示。
- 优化设置页可靠性，置顶、自启动和恢复默认支持执行中禁用、失败回滚和启动重放。
- 修复安装后白屏风险，允许 Tauri IPC 使用 `ipc:` 和 `http://ipc.localhost`，降低 WebView2 CSP 拦截内部调用的风险。
- 增加本地设置清洗，升级后会过滤异常主题、透明度、颜色和字体配置。
- 浅色模式下股票详情页买卖五档、图表、K 线和 tooltip 做了颜色适配。

## 功能说明

### 股票自选

- 支持按股票名称或代码搜索。
- 支持添加、删除、刷新、拖拽排序和右键排序。
- 支持 30 秒自动刷新行情。
- 自动刷新如果遇到接口空响应或偶发失败，会保留旧行情，不会把列表清空。
- 列表展示股票名称、代码、涨跌幅、现价和分时缩略线。
- 点击股票卡片打开详情页。
- 详情页展示分时图、K 线、均线、成交指标、大盘指数和买卖五档。
- 右键股票可录入成本价和股数。
- 录入持仓后，鼠标悬停到卡片会展示市值、成本、盈亏和盈亏比例，盈亏按颜色区分。
- 股票模式下只要存在有效持仓，顶部显示账户资金、当日预计收益和账户总收益。

### 基金自选

- 仅支持场外开放式基金。
- 不支持 ETF、LOF、场内基金、封闭式基金，也不使用股票行情接口里的基金结果。
- 支持按基金名称或代码搜索。
- 支持添加、删除、刷新、拖拽排序和右键排序。
- 列表展示基金名称、代码、估算涨跌幅和估算净值。
- 基金第一版只做列表估算展示，不提供基金详情页、历史净值图和 K 线。
- 右键基金可录入持有金额和收益。
- 录入持仓后，鼠标悬停到卡片会展示持有金额、成本、盈亏和盈亏比例，盈亏按颜色区分。
- 基金模式下只要存在有效持仓，顶部显示账户资金、当日预计收益和账户总收益。

### 股票 / 基金切换

- 顶部标题是切换入口。
- 股票模式标题为 `A+Stock Assistant`。
- 基金模式标题为 `A+Fund Assistant`。
- 股票和基金分开管理，不混合自选列表。
- 切换到基金时会关闭股票详情页，避免基金列表和股票详情混在一起。

### 列表排序与交互

- 股票和基金都支持左侧手柄拖拽排序。
- 拖动启动阈值为 6px，减少误触。
- 拖动时会高亮目标位置。
- 卡片点击、拖拽手柄和删除按钮互不抢事件。
- 删除按钮只在悬停、焦点或选中状态出现。
- 右键菜单保留置顶、上移、下移、移到底部和持仓录入。

### 设置与窗口

- 支持深色、浅色和自定义主题。
- 支持窗口背景透明度调节。
- 支持自定义背景色、字体颜色和字体。
- 支持显示或隐藏大盘指数。
- 支持显示或隐藏股票分时缩略线。
- 支持最小化到托盘。
- 支持窗口置顶。
- 支持开机自启动。
- 支持恢复默认设置。
- 置顶和自启动操作执行中会禁用开关，避免快速连点导致状态错乱。
- 系统调用失败时会回滚 UI 状态。
- 恢复默认会先同步关闭置顶和自启动，失败时不会覆盖本地设置。
- 应用启动后会重新应用置顶状态，保证设置界面和窗口实际状态一致。
- 启动时会清洗旧版本或异常写入的本地设置，降低升级后的透明窗口、异常主题和白屏风险。

## 数据源说明

当前版本使用公开接口获取行情和搜索数据：

- 股票行情：腾讯行情相关接口
- 股票分时和 K 线：腾讯 / IFZQ 相关接口
- 股票搜索：新浪搜索建议接口
- 基金搜索：东方财富基金搜索接口
- 基金估算：天天基金 `fundgz.1234567.com.cn/js/{code}.js`

这些接口不由本项目维护，返回结构、可用性、频率限制和估算更新时间都可能变化。基金实时更新按开放式基金估算净值和估算涨跌幅理解，不等同于交易所逐笔价格。接口异常时会尽量保留旧数据，单条基金刷新失败不会阻塞其他基金刷新。

## 安装与升级

1. 获取 `A+助手_1.0.2_x64-setup.exe`。
2. 双击安装包。
3. 默认优先安装到 `D:\A+助手`。
4. 如已安装旧版本，安装器会显示中文升级提示。
5. 默认保留自选、排序、设置和持仓数据。
6. 卸载时只有主动勾选删除数据，才会清理本地应用数据。

## 常见问题

### 安装后出现白屏怎么办？

`1.0.2` 已修复已知的 WebView2 CSP 拦截 Tauri IPC 风险，并增加本地设置清洗。极端情况下如果仍出现白屏，可以尝试：

- 关闭所有 `A+助手` 进程后重新启动。
- 重新安装最新安装包。
- 清理 `%LOCALAPPDATA%\com.stock.widget\EBWebView` 后再启动。

### 股票列表为什么之前会偶尔变成暂无数据？

旧版股票刷新会用接口返回结果重建整份行情缓存。如果某次自动刷新接口偶发返回空数组，列表会短暂变空。最新版本已改为保留当前自选的旧行情，只有成功返回的新数据才覆盖旧数据。

### 基金涨跌幅为什么有时显示 `--`？

基金估算依赖天天基金估算接口。如果接口未返回估算涨跌幅、估算暂停或数据源异常，列表会显示 `--`，并保持中性颜色。

### 为什么不支持 ETF / LOF？

当前基金功能只面向场外开放式基金。ETF、LOF、场内基金和封闭式基金交易逻辑与数据口径不同，暂不纳入基金第一版。

### 数据是否用于交易？

不会。本项目只做行情展示、自选维护和本地持仓盈亏估算，不提供交易能力，也不构成投资建议。

## 开发环境

- Node.js 18+
- Rust stable
- Windows 10 / Windows 11
- WebView2 Runtime

安装依赖：



## 维护说明

- 本项目只提供行情展示和自选维护，不提供交易能力。
- 股票和基金数据仅用于信息展示和技术研究，不构成投资建议。
